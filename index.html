git实操

发表于 2022-10-16

基本操作

workspace 当前的目录–工作区

Remote 远程仓库

Repository 本地仓库（本地的git项目）

Index 暂存区

pull 拉变更

push 推变更到远程仓库

git clone

git clone 仓库地址

clone项目到本地

git status

查看当前的状态：当前的文档变更状态（新增 修改 untrunk等）

git add

添加一个或多个文件到暂存区

git add 1.txt 2.txt 添加特定文件

git add . 添加当前目录下的所有变更

git commit

提交暂存区的内容到本地仓库

git commit -m "提交说明"

git log

当前分支的历史提交记录，不包括已经被删除的 commit 记录和reset的操作

git log --pretty=online 只会显示版本号和提交时的备注信息

git blame

以列表形式查看指定文件的历史修改记录

git blame [file]

git push

提交本地仓库变更到远程仓库

git push <远程主机名> <本地分支名>:<远程分支名>

git push <远程主机名> <本地分支名> 本地分支与远程分支同名，可以这样操作

git reset

git reset [<mode>] [<commit>]

mode:（一些不常用的后期再完善）

  • –soft

git reset --soft HEAD^

回退时保留工作区，并把回退带来的差异add至暂存区（add了但是没commit）

  • –mixed

git reset --mixed HEAD^

回退时清空暂存区并保留工作区（没有add）

  • –hard

git reset --soft HEAD^

回退的同时重置暂存区和工作区（清空回退带来的变更）

  • –merge
  • –keep
  • –[no-]recurse-submodules

git reset --hard HEAD^

版本说明：

HEAD 当前版本

HEAD^ 上一版本

HEAD^^ 上上一版本

…以此类推

HEAD~0 当前版本

HEAD~1上一版本

HEAD~2 上上一版本

…以此类推

git pull

git pull <远程主机名> <远程分支名>:<本地分支名>

将远程的代码拉取到本地并合并

git reflog

可以查看所有分支的所有操作记录信息（包括已经被删除的 commit 记录和 reset 的操作记录）

分支操作

git checkout

切换分支或者恢复工作区文件（当工作区文件被删除时可以从暂存区恢复到工作区）

git checkout file 恢复暂存区中file文件至工作区

git checkout branch-name 切换到特定分支

git checkout -b branch-name 创建然后切换到特定分支

git branch

git branch new-branch 创建本地新分支

git branch -a 查看所有分支(含远程分支)

git branch 查看本地分支、

git branch -d branch-name 删除本地分支(-D也可以)

git push origin --delete branch-name 删除远程分支

git merge

分支合并

git merge branch-name 把其它分支的变更合并到当前分支

git merge --abort 合并后如果有冲突则取消合并

分支策略

Git鼓励你使用分支完成某个任务，合并后再删掉分支，这和直接在master分支上工作效果是一样的，但过程更安全。

master分支是最稳定的，只能用于发布新版本，平时不能在上面进行开发，要在新建的分支上进行开发，比如dev，这时dev是不稳定的，到产品1.0发布时，将dev分支和master分支合并，在master分支上发布1.0版本。

进阶操作

git stash

将本地没提交的内容(git commit的内容不会被缓存 但git add的内容会被缓存)进行缓存并从当前分支移除，缓存的数据结构为堆栈，先进后出

git stash list 返回缓存的列表

git stash pop 将堆栈中的最新内容应用到当前分支，并删除缓存堆栈中的记录

git remote

git remote 远程库的信息

git remote -v 远程库的详细信息

git rebase

rebase 命令其实关键在于理解「基」，git rebase <基分支>，就是将当前基分支与当前分支的差异提交获取到，然后在「基分支」最新提交点后面将差异提交逐个再次提交，最后将当前分支的 HEAD 指针指向最新的提交点

常用命令

netstat -nultp

列出正在监听的Socket，配合grep 端口号 筛选出指定端口的应用监听是否启动成功

各列的几个说明：

  • 0.0.0.0:22 表示监听服务器上所有ip地址的22端口(0.0.0.0表示本地所有ip)

  • :::22 这个也表示监听本地所有ip的22端口，跟上面的区别是这里表示的是IPv6地址，上面的0.0.0.0表示的是本地所有IPv4地址

  • Local Adress分为0.0.0.0所有ip都可访问，127.0.0.1只有本机可访问

sort

参考：https://www.cnblogs.com/51linux/archive/2012/05/23/2515299.html

sort将文件的每一行作为一个单位，相互比较，比较原则是从首字符向后，依次按ASCII码值进行比较，最后将他们按升序输出。

参数

awk '{print $1}' access.log |sort|uniq -c|sort -nr|head -10

concurrentHashMap

发表于 2022-03-22 更新于 2022-03-23 分类于 jdk源码

类注释

检索操作不会受并发导致阻塞，对于一些聚合操作(putAll/clear)，并发访问可能只会反应出一部分的更新效果。

迭代器中的修改不会发生ConcurrentModificationException，但是某个特定时刻只会允许一个线程去修改。一些聚合状态的方法size、isEmpty、containsValue只有当不存在受其它线程的并发更新影响时才有用。其结果只能反映瞬时状态，但是对于监控或评估足够了，但不适合程序控制。在构造对象实例时提供一个预估的size是个好的选择，避免过多的resize和rehash（相对比较耗时的操作）
